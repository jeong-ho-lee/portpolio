1-3) Perceptron model 구현¶

이 부분에서는 직접 Perceptron model을 구현해 봅니다.

Perceptron model은 생성자, forward, backward, train, evaluation 다섯개 함수로 이루어져 있습니다.

  • __init__ 생성자에서는 Perceptron의 weights와 bias를 초기화합니다.
  • forward에서는 input을 Perceptron의 가중치를 이용해서 예측을 수행합니다.
  • backward에서는 Perceptron의 가중치를 학습하기 위한 gradient를 계산합니다.
  • train에서는 Perceptron을 학습하는 과정으로 forward와 backward를 차례로 반복하여 Perceptron의 가중치를 업데이트 합니다.
  • evaluation에서는 들어온 input data를 학습된 가중치를 이용하여 예측하고 결과를 반환합니다. (주의: Accuracy 계산)

아래 # <your code> 부분을 채워 넣어서 Perceptron class를 직접 작성하여 구현하세요.

세부 구현 사항:

  • weights는 랜덤으로 초기화, bias는 0으로 초기화한다.
  • activation function은 입력값을 기준으로 0 초과는 1, 0 이하는 -1을 출력하게 한다.
  • 주의: **Cost Function은 MSE 또는 Cross-Entropy가 아닌 $J(w) = -y(w^Tx)$를 사용**한다. , $x\in Y$는 틀린 샘플의 집합

class Perceptron():
    def __init__(self, num_features):
        self.weights = [random.random() * 0.001 for i in range(num_features)]
        self.bias = 0
        self.lr = 0.01
        
    # 모델의 출력을 반환하는 함수
    def forward(self, x):
        linear = sum(self.weights[i] * x[i] for i in range(len(self.weights))) + self.bias
        prediction = 1 if linear > 0 else -1
        return prediction
        
    # 손실함수에 기반하여 모델의 가중치를 업데이트하기 위한 gradient를 반환하는 함수
    def backward(self, x, y):
        gradient = sum(y * x[i] for i in range(len(self.weights)))
        return gradient
        
    def train(self, x, y, epochs):
        # epochs 만큼 학습
        for e in range(epochs):
            # Each data point (Stochastic learning)
            for i in range(len(y)):
                x_, y_ = x[i], y[i]
            
                # 오분류 샘플에 대해서만 gradient를 계산하여 모델의 가중치를 업데이트 함
                if y_ != self.forward(x_): 
                    for j in range(len(x_)):
                        gradient = self.backward(x_, y_)

                        self.weights[j] += self.lr * gradient 
                         
                    self.bias += self.lr * (y_ - self.forward(x_))
                    
    def evaluate(self, x, y):
        accuracy = sum(1 if self.forward(x[i]) == y[i] else 0 for i in range(len(y))) / len(y)
        return accuracy

S2. Discussion¶

1) 예시에서 사용된 activation 함수 이외의 어떤 함수가 있는지 찾아보고 설명해보세요.

[답변작성] 예시에서 사용된 계단 함수 외에도 로지스틱 시그모이드, 하이퍼볼릭 탄젠트, 소프트플러스, 렉티파이어 등의 활성 함수가 있다. 가장 먼저 로지스틱 시그모이드는 1 / (1 + exp(-a * s))의 형태로 표현되며 (0, 1)의 범위를 가진다. 1차 도함수는 a * (1 / (1 + exp(-a * s))) * (1 - 1 / (1 + exp(-a * s)))이다. 다음으로 하이퍼볼릭 탄젠트는 2 / (1 + exp(-a * s)) - 1의 형태로 표현되며 (-1, 1)의 범위를 가진다. 1차 도함수는 a / 2 * (1 - (2 / (1 + exp(-a * s)) - 1) ** 2))이다. 로지스틱 시그모이드와 하이퍼볼릭 탄젠트는 입력값이 점점 커짐에 따라 미분값이 0에 점점 가까워지는 특징이 있다. 또, 소프트플러스는 log(1 + exp(s))의 형태로 표현되며 (0, np.inf)의 범위를 가진다. 1차 도함수는 1 / (1 + exp(-s))이다. 렉티파이어는 max(0, s)의 형태로 표현되며 [0, np.inf)의 범위를 가진다. 1차 도함수는 0 if s < 0 1 elif s > 0 else NULL이다. 소프트플러스와 렉티파이어는 입력값이 점점 작아짐에 따라 미분값이 0에 점점 가까워지거나 0이 되는 특징이 있다.

2) Perceptron 하나로 풀 수 없는 문제는 어떤것이 있는지 왜 그런지 설명해보세요.

[답변작성] 단층 퍼셉트론은 XOR 문제를 해결할 수 없다. XOR 문제는 x = [[0, 0], [0, 1], [1, 0], [1, 1]], y = [1, -1, -1, 1]과 같은 형태로 선형 결정 직선으로는 절대 정확도 100%로 입력 데이터를 이진 분류하지 못한다. XOR 문제를 풀기 위해서는 다층 퍼셉트론을 이용하여 은닉층을 통해 특징 공간을 변환하여 선형 결정 직선으로 데이터를 이진 분류하는 것이 가능하도록 만들어 해결해야 한다.
