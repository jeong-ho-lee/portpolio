수업

DB란? 어떤 조직에 하나의 데이터베이스가 있고 여러 명의 사용자가 공유해서 사용한다
사용자나 조직마다 사용하는 데이터가 다르다 - 학교라고 하면 학생이 쓰는 데이터가 있고 교수가 쓰는 데이터가 있다 근데 이런 데이터가 통합되어 있음 - 사용자가 원하는 데이터만 골라서 씀
그럼 통합된 데이터를 어떻게 넣냐? (정리하냐?) - 체계적으로 구조를 잡고 저장을 한다

정보와 데이터의 차이점? - 데이터는 날 것 정보는 가공된 것 - 즉 내가 필요한 것은 정보
나한테는 정보인게 다른 사람에게는 데이터일 수 있고 반대도 가능 질의에 의해 발생하는 현상

중복의 최소화 - 만약 데이터가 중복된다면 데이터의 수정이 발생했을 떄 다른 정보가 발생해버리게 된다
메타데이터란? - 데이터에 대한 설명 예) 나이 데이터는 0 - 100 사이의 양의 정수이다
데이터 독립성? - 응용 시스템과 분리 - 둘 중에 하나가 수정되어도 영향을 끼치지 않음
효율적인 데이터 접근 - 대규모 - 검색? = 질의 무엇을 원하는지만 말하면 알아서 빠른 방법으로 찾는다
백업과 회복 - 문제가 발생했다면 문제가 발생하기 이전의 상황으로 되돌린다 - 회복

데이터 무결성 - 조건에 맞지 않는 값을 저장하지 않는다 - 유일한 값 (학번 같은)이 중복이 발생했다 - 엔티티 무결성
